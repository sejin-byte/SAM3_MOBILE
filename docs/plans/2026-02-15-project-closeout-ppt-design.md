# SAM3_MOBILE 프로젝트 종료 발표자료 설계서

- 작성일: 2026-02-15
- 목적: 모델 내부 아키텍처/원리 + 프로젝트 배경/진행 과정/성과를 한 번에 설명 가능한 발표자료(PPT) 제작
- 형식: 한국어, 15장 이상, 혼합형(비기술 -> 기술 심화)

## 1. 아키텍처(슬라이드 구조)
총 17장 구성

1) 표지
2) 프로젝트 한 줄 요약
3) 왜 이 프로젝트를 했는가(배경)
4) 목표/KPI
5) 전체 진행 타임라인
6) 최종 상태 요약
7) 모델 아키텍처 개요(상위 블록)
8) 데이터 흐름(입력->텍스트/비전->디코더)
9) 지식 증류 원리(Teacher-Student)
10) 학습 단계(Phase1/Phase2/Video/Quant)
11) 핵심 코드 변경사항
12) 재학습 정량 성과
13) Teacher 대비 위치
14) 양자화 결과 및 해석
15) 정성 시각 결과(이미지 포함)
16) 배포/운영 관점 정리
17) 결론 및 다음 사이클 제안

## 2. 구성요소
- 생성 스크립트: `scripts/build_project_closeout_ppt.py`
- 산출물: `docs/ppt/SAM3_MOBILE_Project_Closeout_20260215_ko.pptx`
- 보조 문서: `docs/ppt/README.md`
- 입력 이미지: `docs/visual_qa_*.png`

## 3. 데이터/콘텐츠 출처
- 재학습 영향: `outputs/reports/retrain_impact_20260215.md`
- teacher 비교: `outputs/reports/teacher_vs_student_miou_tiny_20260215_165149.json`
- 양자화 로그: `logs/quantization/compare_20260215_161405.log`
- 종료 리포트: `docs/RETRAINING_PERFORMANCE_ANALYSIS_20260215.md`

## 4. 오류 처리/안정성
- 이미지가 없으면 경고 텍스트 박스를 넣고 계속 생성
- 출력 디렉터리 자동 생성
- 스크립트 실행 실패 시 경로/의존성(`python-pptx`) 안내

## 5. 검증 계획
1) 스크립트 실행으로 PPT 파일 생성 확인
2) 생성 파일 존재/크기 확인
3) 슬라이드 개수(17장) 확인
4) 핵심 이미지 포함 여부 확인
5) Git 커밋/푸시 확인

## 6. 구현 범위
- 구현 포함: 콘텐츠 생성 자동화, 슬라이드 텍스트/도형/이미지 배치, 결과 파일 생성
- 구현 제외: 테마 디자이너 수준의 수동 마감, 애니메이션/고급 템플릿
